# 消融草稿（Day3 · 样本轨迹）

## 实验设计

- **变量**：system-prompt（有 / 无）
- **固定项**：任务集（read-config + list-dir）、模型（deepseek-v4-flash）、工具集（read/bash）
- **数据**：每组 2 条样本轨迹

## 结果

| 组别 | 成功率 | 平均 token |
|------|--------|-----------|
| 有 system-prompt | 1.00 | 335 |
| 无 system-prompt | 0.00 | 128 |

## 归因

无 system-prompt 时 agent 不知道 `<tool_call>` 约定 → 从不调工具 → 全部失败。
有 system-prompt 的 token 消耗更高（多了工具使用说明），但换来了任务完成能力。

## 局限

- 样本量太小（各 2 条），统计上不显著
- 样本是人工构造的，无法反映真实 agent 的行为分布
- 未做多次运行取均值（无法量化方差）

## D4 起改进方向

- 用真 agent 轨迹替换构造样本，每组 ≥10 条
- 每组多次运行取均值 ± 标准差
- 增加更多对照维度（温度、上下文窗口大小、工具数量等）
