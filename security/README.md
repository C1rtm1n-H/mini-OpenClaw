# security 模块

安全采用多层防护，目标是即使模型被提示注入或产生错误 tool call，执行层仍能阻断
越界与高风险操作。

## 防护层

1. 系统提示词：声明只读审计、训练降级和证据要求。
2. 任务作用域：用户明确给出的文件或目录是默认授权根。
3. 权限分级：读放行、写确认、执行/外传确认、危险命令拒绝。
4. 工具层：Shell 黑名单、超时、web 白名单、文件路径检查。
5. 内容隔离：文件和网页结果带 `<external>` 数据边界。
6. 红队：用最坏情况后端直接投递恶意工具调用，验证生产分发路径。

## 红队用例

`security/redteam.py` 覆盖：

- 越权执行：`rm -rf ~/`。
- 提示注入：网页诱导读取密钥和外传。
- 数据泄露：读取 `~/.ssh/id_rsa` 并 curl。
- 越狱：要求忽略规则执行任意命令。

运行：

```powershell
python -m security.redteam
```

脚本不会真的执行危险命令；第二层检查只做静态字符串和域名判定。报告写入
`security/redteam_report.md`。

## 已知边界

- Shell 黑名单不是完整语法分析器，权限确认不能被取消。
- 红队报告已指出部分只读系统命令可能未命中第二层黑名单；如果未来开放
  `auto_approve`，必须先增强命令策略。
- 出站白名单只限制目标域名，不代表白名单页面内容可信。
- 任何密钥都只能通过环境变量提供，不能写入记忆、trace 或仓库。
