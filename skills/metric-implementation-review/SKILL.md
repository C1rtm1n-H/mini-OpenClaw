# 指标实现审查

## 流程

1. 用 `read` 提取论文、README 或配置中对指标的名称、公式、平均方式、阈值、单位和评估数据说明。
2. 用 `grep` 搜索指标函数、调用位置、预测后处理和结果聚合，再用 `read` 阅读完整上下文。
3. 建立“文档声明 → 代码实现 → 输出字段”映射，逐项核对名称、方向、输入、阈值和聚合方式。
4. 检查 `model.eval()`、`no_grad/inference_mode`、checkpoint、macro/micro/weighted、batch 加权、分布式聚合和测试集调参。
5. 仅在存在小型确定性样例时运行低成本验证；否则给出最小单元测试方案，不声称动态行为已确认。
6. 将发现标为“已确认、疑似、未验证”，记录文件和行号，并输出影响与修复建议。

## 重点风险

- 对 batch 指标做未加权平均。
- README 写 F1，代码实际报告 accuracy。
- 在 test set 上选阈值或最佳 checkpoint。
- 多卡只统计单卡结果。
- 缺少 eval mode 或梯度关闭。

## 输出格式

```markdown
# 指标实现审查
| 声明 | 实现位置 | 一致性 | 状态 | 证据 |
## 高风险差异
## 最小验证方案
## 未确认事项
```
