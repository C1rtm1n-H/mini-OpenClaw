# 实验复现计划

## 流程

1. 用 `glob`、`grep`、`read` 收集 README、依赖、配置、数据说明、训练入口、评估入口和已有审计报告。
2. 区分已确认要求、文档声称、合理推断和缺失信息；缺失信息转成前置问题，不擅自补齐。
3. 将复现拆成环境、数据、最小烟雾测试、小规模运行、完整运行、评估和结果核对七个阶段。
4. 为每阶段列出输入、命令草案、资源、成功标准、失败停止条件、产物和回滚方式。
5. 默认只生成计划，不执行安装、下载、训练、评估、上传或覆盖操作；需要执行时逐阶段请求授权。
6. 设置由低到高的成本门槛：前一阶段未通过，不进入后一阶段。
7. 输出计划、风险、资源缺口、验证矩阵和最终复现判定标准。

## 计划原则

- 先 CPU/小数据/少 step，再 GPU/全量数据。
- 固定并记录 seed、配置、环境、数据版本和 git commit。
- 保留 stdout、指标、预测、checkpoint 和配置快照。
- 不根据代码猜测精确运行时间或显存；没有证据时给待测项。

## 输出格式

```markdown
# 实验复现计划
| 阶段 | 输入 | 命令草案 | 成功标准 | 停止条件 | 产物 |
## 资源与授权点
## 风险和回滚
## 复现完成判定
```
