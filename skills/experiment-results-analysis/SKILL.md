# 实验结果分析

## 流程

1. 用 `glob` 定位结果文件，用 `read` 检查字段、单位、指标方向、配置标识和随机种子；字段含义不明时先标记缺口。
2. 检查缺失值、重复运行、失败运行、样本数不一致和不可比较的设置。不要把不同数据集、划分或指标直接混合。
3. 仅在环境已有合适库时，用 `bash` 运行短小统计命令，计算每组样本数、均值、标准差、最小值和最大值；不安装依赖。
4. 比较方法时固定数据、指标和实验条件，报告绝对差值与相对变化，并注明“越高越好”或“越低越好”。
5. 分析消融时逐项对应变化因素；没有重复运行时，不声称结果稳定或具有统计显著性。
6. 输出数据质量、汇总表、主要发现、异常点、可支持结论和不可支持结论。

## 注意事项

- 不补造缺失 seed、方差、置信区间或实验数值。
- 日志中的中间指标不能替代最终指标。
- 统计显著性结论必须有检验方法、样本量和结果支持。

## 输出格式

```markdown
# 实验结果分析
## 数据质量
## 汇总结果（mean ± std，注明 n）
## 最佳配置与相对变化
## 异常运行
## 可支持的结论
## 不可支持的结论与待补实验
```
