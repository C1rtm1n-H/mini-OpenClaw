# 论文速读

1. 用户明确指定单个论文文件时，只处理该文件及其同名 TXT 缓存，不读取同目录其他内容，也不扫描 Agent 宿主仓库；只有用户给出目录时才在该目录内 `glob`。已有 TXT 时优先复用；只有 PDF 时调用 `pdf_extract`。缓存过期时不要擅自设置 `overwrite=true`，除非用户明确要求刷新。
2. 按“研究问题、核心贡献、方法、数据与实验、主要结论、局限性”提取证据；为关键结论保留页码、章节或原文位置。
3. 区分作者明确陈述与自己的推断；无法从材料确认的内容标为“未说明”。
4. 输出结构化速读报告，并给出值得复查的图表、公式或实验设置。

## 注意事项

- 论文解析失败时说明受限原因，并请求文本版或可复制内容。
- 用户只提供论文、未提供关联代码仓库时，输出基于论文证据的复现计划并明确代码缺口；不要拿当前工作目录中的其他工程代替论文仓库。
- 只有论文时不执行 Python 版本、依赖 import、训练入口等代码环境验证；这些检查标为“不适用：未提供代码仓库”，不要创建 blocked TODO。
- 用户需要文件产物时，可将命名明确的 Markdown 报告写到当前工作目录，例如 `<论文名>-reproduction-report.md`。

## 示例输出

```markdown
# 论文速读：<题名>
- 研究问题：...
- 核心贡献：...
- 方法：...
- 数据与指标：...
- 主要结论：...（第 4.2 节）
- 局限性：...
- 待复查：图 3 的消融设置未说明随机种子。
```
