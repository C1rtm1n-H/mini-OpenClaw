# Baseline 比较

## 流程

1. 确认比较目标、任务、数据集、指标和约束；不比较任务或协议不一致的结果。
2. 用 `glob`、`read` 收集各 baseline 的 README、配置、结果和审计报告，统一名称和字段。
3. 按文档完整性、依赖、数据、seed、指标、checkpoint、日志、资源和许可证建立比较矩阵。
4. 区分论文报告性能、仓库可复现证据和本地验证结果，不把论文数值视为代码已复现。
5. 对缺失信息、不同数据划分或不同指标协议设置不可比标记，不强行排名。
6. 按用户约束给推荐，并说明权衡、阻塞项、最小验证实验和替代选择。

## 评分原则

- 评分必须来自明确维度和证据，不只根据最终指标。
- 没有资源证据时不估造显存和时间。
- 性能、复现成本、工程风险和许可证分开报告。

## 输出格式

```markdown
# Baseline 比较
| Baseline | 性能证据 | 复现难度 | 资源 | 风险 | 可比性 |
## 关键差异
## 推荐与权衡
## 最小验证计划
## 缺失信息
```
