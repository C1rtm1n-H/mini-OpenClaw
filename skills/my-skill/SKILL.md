# 科研辅助

先确认输入文件、研究问题和期望产物；信息不足时明确缺口，不猜测论文内容、引用信息或实验结果。根据任务选择下列流程，可组合执行。

## 论文速读

1. 用 `glob` 定位 PDF、Markdown 或文本文件；用 `read` 读取文本。若只有 PDF，先用 `bash` 调用环境中已有的 PDF 文本提取工具，再读取提取结果。
2. 按“研究问题、核心贡献、方法、数据与实验、主要结论、局限性”提取证据；为关键结论保留页码、章节或原文位置。
3. 区分作者明确陈述与自己的推断；无法从材料确认的内容标为“未说明”。
4. 输出结构化速读报告，并给出值得复查的图表、公式或实验设置。

## 文献整理

1. 用 `glob` 收集文献文件，用 `read` 提取题名、作者、年份、研究问题、方法、数据集、指标、结论和局限。
2. 统一术语与字段；缺失字段写“未说明”，不补造 DOI、引用量或实验数值。
3. 按用户的研究问题建立对比表，再归纳共识、分歧、研究空白和可延伸方向。
4. 需要落盘时，用 `write` 生成 Markdown 报告；写入前先确认输出路径不会覆盖重要文件。

## 实验脚本

1. 用 `glob`、`grep` 和 `read` 查找入口脚本、配置、依赖、数据路径、随机种子、输出目录与评价指标。
2. 先用 `bash` 执行帮助、语法检查或最小样例；确认命令和资源需求后再运行完整实验。
3. 保存命令、关键参数、环境信息、退出码和指标。失败时定位首个根因错误，给出最小修复建议后重新验证。
4. 比较实验时只改变目标变量，整理运行条件与结果表；不要把日志中的中间值当作最终指标。

## 注意事项

- 论文解析失败时说明受限原因，并请求文本版或可复制内容。
- 运行未知脚本前先检查其参数与副作用；不执行删除、上传、安装或长时间高资源任务，除非用户明确授权。
- 引用结论时附可追溯位置；报告中的推断使用“可能”“表明”等措辞。

## 示例输出

```markdown
# 论文速读：<题名>
- 研究问题：...
- 核心贡献：...
- 方法：...
- 数据与指标：...
- 主要结论：...（第 4.2 节）
- 局限性：...
- 待复查：图 3 的消融设置未说明随机种子。
```
