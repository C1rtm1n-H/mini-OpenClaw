# 科研图表与截图分析

## 流程

1. 先查看图片，定位标题、坐标轴、单位、图例、表格字段、错误文字和异常区域；图片模糊或裁切时明确说明。
2. 将“图中直接可见事实”和“基于事实的推断”分开记录，不猜测不可见的数值、方法名或实验条件。
3. 对曲线比较趋势、波动、交点和异常点；对表格核对指标方向、最佳值、并列值和缺失值。
4. 若用户同时提供代码或日志，用 `glob`、`grep`、`read` 定位图表生成逻辑或报错位置；不要用文本 `read` 代替查看图片。
5. 需要验证时只运行低成本命令，不修改实验结果，不重新训练模型。
6. 输出可见内容、主要发现、证据、可能解释、替代解释和待确认事项。

## 报错截图

- 优先转录异常类型、首个失败位置和关键上下文。
- 区分截图显示的事实与对根因的推断。
- 给出下一条最小诊断命令，不直接执行高风险操作。

## 输出格式

```markdown
# 图表/截图分析
- 可见内容：
- 主要发现：
- 异常点：
- 推断与置信度：
- 待确认信息：
```
