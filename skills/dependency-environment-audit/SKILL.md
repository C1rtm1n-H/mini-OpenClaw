# 依赖与环境审计

## 流程

1. 用 `glob` 定位 `requirements*.txt`、`environment*.yml`、`pyproject.toml`、`setup.py`、Dockerfile 和安装文档。
2. 用 `grep`、`read` 收集源码 import、动态 import、系统命令、本地路径和 CUDA/GPU 检查逻辑。
3. 对照“代码使用的依赖”和“声明的依赖”，标记未声明包、疑似无用包、过宽/冲突版本及私有包。
4. 检查 Python、操作系统、编译器、CUDA、驱动、GPU、系统库和外部命令是否有明确要求。
5. 仅运行版本查询或包存在性等低成本检查；不执行安装、升级、环境创建或网络查询，除非用户授权。
6. 区分项目缺陷与当前机器环境差异，记录证据和验证状态。
7. 输出环境矩阵、依赖差异、跨平台风险和推荐的最小环境规范。

## 注意事项

- 标准库、项目内模块和第三方包要分开。
- import 名与包名可能不同，映射不确定时标为待确认。
- 不把当前机器已安装视为项目已正确声明。

## 输出格式

```markdown
# 依赖与环境审计
## 环境要求矩阵
## 未声明/冲突依赖
## 系统与 GPU 风险
## 当前机器验证
## 建议的环境规范
```
