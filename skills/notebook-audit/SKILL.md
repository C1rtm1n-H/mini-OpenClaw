# Notebook 审计

## 流程

1. 用 `glob` 定位 `.ipynb`，用 `read` 查看其 JSON 结构；文件过大时用 `grep` 定位关键单元，不一次加载全部输出。
2. 按单元顺序提取 import、变量定义、数据路径、随机种子、训练/评估调用和保存逻辑。
3. 检查执行计数乱序、引用未在前文定义的变量、依赖交互状态、重复定义、隐藏参数和未保存配置。
4. 检查硬编码路径、安装/下载命令、密钥、超大输出、数据划分、测试集使用和随机性。
5. 默认不执行 Notebook。需要验证时先提出只读解析或最小单元方案，并获得用户授权。
6. 输出单元证据、复现风险、建议执行顺序和脚本化迁移建议。

## 重点风险

- 执行计数与文件顺序不一致。
- kernel 重启后无法从头运行。
- 中间变量依赖未记录。
- Notebook 内含 `%pip`、下载、删除或密钥。
- 训练、评估和可视化共享可变状态。

## 输出格式

```markdown
# Notebook 审计
## 执行与状态问题
## 依赖、数据和随机性
## 安全与输出风险
## 从头复现步骤
## 脚本化建议
```
